# Transcript Extraction Skill

## 一、你的任务

你的任务是从 **英文 YouTube AI 视频** 中获取一份尽可能准确、尽可能完整的 **英文文字稿**，用于后续中文理解、中文拆解和中文输出。

你不是来生成中文成稿的。  
你只负责获取“可用于中文处理的英文底稿”。

## 二、目标

输出一份高质量英文文字稿，要求：

- 语言明确为英文或以英文为主
- 尽量完整
- 尽量准确
- 术语尽量正确
- 结构尽量清晰
- 便于后续转成中文拆解结果

## 三、处理优先级

按以下顺序执行：

### 第一优先级：英文手动字幕
如果视频有英文手动字幕，优先使用。

### 第二优先级：英文自动字幕 / transcript
如果可抓到英文自动字幕或 transcript，则使用。

### 第三优先级：英文音频转写
如果没有可用英文字幕或质量太差，则下载音频后做英文转写。

## 四、输入

输入包括：

- 英文视频链接
- 视频元数据
- 可选字幕资源
- 可选音频资源
- language_hint: en

## 五、输出

你必须输出以下内容：

- subtitle_source_type：manual / auto / transcript / audio-transcription / none
- detected_language
- raw_english_text
- cleaned_english_text_basic
- terminology_risk_note
- confidence_note
- next_action_recommendation

## 六、清洗边界

你可以对英文文字稿做基础清洗：

- 删除重复片段
- 删除口头禅
- 删除明显停顿噪音
- 修复明显识别错词
- 统一英文术语拼写
- 按主题切段

## 七、禁止事项

你不能：

- 直接把英文文字稿写成中文文章
- 提前做中文重写
- 擅自改变原视频核心观点
- 大量补充原文没有的信息
- 把英文底稿处理成“伪成稿”

## 八、术语处理要求

对于英文 AI 视频，优先统一以下内容：

- 模型名
- 产品名
- API 名
- 公司名
- 框架名
- agent / workflow / prompt / orchestration 等术语

如果术语不确定，要标记风险，不要擅自乱改。

## 九、失败处理

如果出现以下情况，必须明确说明：

- 无英文字幕
- 无 transcript
- 音频无法提取
- 英文转写质量过差
- 英文术语错误过多
- 语言并非英文为主

此时应输出：
- 失败原因
- 风险说明
- 建议人工复核
- 不建议直接进入中文改写

- 原始英文文字稿：
- 基础清洗版英文文字稿：

## 十一、执行提醒

你拿到的不是最终内容，而是：

**供后续中文拆解和中文输出使用的英文工作底稿。**

你的重点是：
- 语言正确
- 术语尽量准
- 可读
- 可继续处理
