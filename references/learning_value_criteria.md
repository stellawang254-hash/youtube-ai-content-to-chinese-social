# Learning Value Criteria

## 一、这份文件的作用

这份文件用于判断：

**一个 YouTube AI 视频，是否真的值得进入学习和沉淀流程。**

它解决的问题是：  
很多视频看起来很火、很新、很热闹，但看完之后没有真正留下什么，也很难迁移到自己的项目里。

所以，这份标准的目标是：

**把“看起来有意思”的视频，和“真正值得学”的视频区分开。**

## 二、总判断原则

一个值得优先处理的视频，至少要满足以下三点中的两点：

1. 能学到方法  
2. 能学到流程  
3. 能迁移到自己的项目或 skill 设计中  

如果三点都不满足，通常不建议优先做。

## 三、高学习价值视频的特征

### 1. 有明确问题
视频不是在泛泛而谈，而是在解决一个具体问题。

### 2. 有清晰步骤
视频里不是只讲“是什么”，还讲“怎么做”。

### 3. 有结构
视频内容不是散讲，而是有框架。

### 4. 有实操演示
视频最好包含：
- 实战案例
- 系统搭建过程
- 工具调用过程
- 工作流搭建过程
- 项目演示

### 5. 有迁移价值
你看完后，能问自己：
- 这个方法能不能用到我的项目里
- 这个结构能不能沉淀成一个 skill
- 这个流程能不能改造成工作流
- 这个案例能不能变成我自己的练手方向

### 6. 有可复用的方法
视频不是一次性技巧，而是有可重复使用的逻辑。

## 四、中高学习价值视频的特征

这类视频不一定特别硬核，但仍然值得看。

比如：
- 某个工具的深入使用经验
- 某个模型的高级用法总结
- 某种 prompt 框架的系统整理
- 一个 workflow 的复盘
- 一个真实项目的经验总结

## 五、低学习价值视频的特征

以下类型视频，默认优先级较低：
- 纯工具盘点
- 纯热点评论
- 纯概念视频
- 结果展示型视频

## 六、推荐评分维度

每个候选视频，可以从以下维度打分：

### 1. 问题清晰度
它到底在解决什么问题？

### 2. 方法密度
里面有没有明确方法？

### 3. 流程清晰度
是不是讲清了步骤和流程？

### 4. 可迁移性
是否能迁移到自己的工作和项目？

### 5. 可实操性
是不是看完就能动手试？

### 6. 可沉淀性
是否值得做成：
- 学习笔记
- skill
- workflow
- 项目方案

## 七、推荐判断句

在判断一个视频值不值得优先学时，优先问自己：

- 看完之后，我能学到一套什么方法？
- 里面有没有我今天就能试的动作？
- 这个视频是让我“知道了”，还是让我“会做了”？
- 它能不能迁移到我的项目里？
- 它值不值得我沉淀成一个 skill 或模板？

## 八、学习优先级分层

### S 级：强烈优先
满足以下多数特征：
- 有明确问题
- 有完整步骤
- 有框架
- 有案例
- 有演示
- 可迁移
- 可实操

### A 级：优先处理
有方法、有结构，虽然不一定非常完整，但对学习有明显帮助。

### B 级：可补充
有一定启发，但更多是补认知、补视角，不一定适合马上动手。

### C 级：低优先
有热度或有趣，但学习价值一般。

### D 级：淘汰
对学习帮助很弱，更多只是热点、盘点、观点或娱乐。

## 九、与发文价值的关系

学习价值高，不一定发文价值高。  
发文价值高，也不一定学习价值高。

本 skill 的原则是：

**先判断学习价值，再判断发文价值。**

## 十、推荐输出字段

在筛选视频时，建议增加以下字段：

- learning_priority：S / A / B / C / D
- learning_reason：
- practical_takeaways：
- reusable_method：
- migration_note：

## 十一、最终提醒

真正值得优先学的视频，不一定是最热的，  
而是你看完之后，能带走一套方法、一种结构，或者一个能马上试起来的动作。
